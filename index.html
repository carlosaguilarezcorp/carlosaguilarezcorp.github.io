Excercise #1
Description

The company X20DF will release his streaming service for all their old and upcoming movies the next year in the US and Mexico. This platform will work as a subscription service like Netflix or Disney+. The service will have multiple subscription plans depending on the number of devices and the country. Below are the subscription plans for the first release:

Name# of devicesPriceCountries available
Individual1$ 9.99US, MX
Family2$ 14.99US, MX
Big Family4$ 19.99US

The subscription plan will charge monthly to customers. If the customer is unable to pay the subscription, the system will charge 5% of the subscription price per day until the customer pays the subscription price + late fee. The customers can cancel the service at any moment but only if the service is already paid for the current month. Also, the customers will be able to upgrade or downgrade their service plan at any moment.

Problem

The first round of user acceptance testing has been not satisfactory, the UAT team found a considerable number of bugs in the system. The main issue is that the system is not charging the right amount after doing other transactions first like upgrading the subscription plan.

Another concern is the lack of documentation for the number of test cases and how they are being organized by the QA team. The names of the test cases don’t follow a convention and are difficult to find in the test case repository. Below are some examples for the name of the test cases:

  • Plan Family Create USA
  • CancelPlanLateFee4

The company decided to change the QA team for the project and that’s why you were brought in. The company requires you to work in the first deliverables detailed in the next section.

Deliverables
  • The naming convention for the test case names.
  • A list with the names of the test cases that you consider necessary to cover at least 90% of the system’s functionality. (Only the name of the test cases)
  • Document how you calculated the testing coverage. You can add any type of diagram, tables, etc.
  • Any document that you consider necessary to understand your logic behind the other deliverables.
Exercise #2
Description

2gO is a taxi company that is designing his first applications to connect customers with his taxi service. For the first release of the application, the customers will be only able to request a taxi.

The application will follow a microservices architecture, a set of microservice APIs will be created to be consumed by the mobile and web applications. The bellow is the list of microservices APIs:

  • Location Service: Provides information about the GPS locations for customers and the taxis.
  • Customer Service: Provides information about the customers.
  • Car Service: Provides information about the taxis.

Your main task as part of the QA team for the design phase is to develop a test plan for the project. The test plan must include all the testing definitions and steps for the system’s components, the test plan details how and when the process should be executed during the project.

Deliverables
  • The test plan for the project. The test plan doesn’t have to include technical details like the tools to use for testing (you can add them if you want to). It should include the scope for testing, testing levels, etc. everything from a concept level. You can use the syllabus for the tester foundation level certification from the ISTQB organization as a guide: Foundation Level Syllabus
